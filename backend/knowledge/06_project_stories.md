# Project Stories

## Project story: Provider license expiry redesign

### Situation

Healthcare operations teams need accurate provider license data. When license status is wrong, stale, or hard to reverse, operations teams lose time and engineers get recurring tickets.

### Task

Ankit worked on redesigning the provider license-expiry workflow to make it more accurate and operationally useful.

### Action

He improved the workflow model, made state transitions clearer, and focused on reversibility and operational correctness.

### Result

The system reduced a recurring class of operational issues and made the workflow easier to manage.

### Best interview angle

This story shows Ankit's ability to solve problems where the hard part is not just coding, but modeling messy real-world operations.

## Project story: Pulley valuation platform

### Situation

### Result

## Project story: Ask Ankit portfolio assistant

### Situation

Most portfolio websites are static. Recruiters often skim quickly and may not find the most relevant information.

### Task

Ankit is building `Ask Ankit`, a portfolio assistant backed by a structured Markdown knowledge base and RAG.

### Action

The system uses topic-specific Markdown files, embeddings, retrieval, and answer generation with citations.

### Intended result

Recruiters and founders can ask focused questions like:

- What kind of engineer is Ankit?
- What did Ankit build at Pulley?
- Is Ankit a fit for an early-stage startup?
- What is Ankit learning in AI?

### Best interview angle

This project shows practical AI product thinking. It is not AI for hype. It uses AI to solve a real communication problem.
